目的
LOD計算の基本的な使い方とLODを利用した場合のフィルターの順番に関して共有したい。
この2つに関する理解は業務上非常に大切である。(この2を用いないプロジェクトを見たことがない程に高頻度に利用する)
またこれらは数値の計算順序に関わるためバグの起因となりやすい。バグのないレポート作りをする上でも大切だと思う。
LOD計算
　
  • LOD計算とは
    「Level of Detail」の略で、「詳細レベル」と訳される。詳細レベルとは、計算のまとまりのことである
    {[FIXED | INCLUDE | EXCLUDE] <ディメンションの宣言 > : <集計式>}
    種類 使い方
    Fixed FIXED 詳細レベルの式は、ビューの詳細レベルに関係なく (つまり、ビューの中の他のいずれのディメンションにも関係なく)、指定されたディメンションを使用して値を計算します。 FIXED 詳細レベルの式も、コンテキスト フィルター、データ ソース フィルター、および抽出フィルター以外のビュー内のすべてのフィルターを無視します。
    例: { FIXED [Region] : SUM([Sales]) }
    include INCLUDE 詳細レベルの式は、ビューに含まれるディメンションに加えて、指定されたディメンションを使用して値を計算します。 INCLUDE 詳細レベルの式は、ビューにないディメンションを含める場合に最も役に立ちます。
    例: { INCLUDE [Customer Name] : SUM([Sales]) }
    exclude EXCLUDE 詳細レベルの式は、式からディメンションを明示的に除外します。つまり、ビューの詳細レベルからディメンションを除去します。 EXCLUDE 詳細レベルの式は、ビューに含まれるディメンションを除外する場合に最も役に立ちます。
    例: {EXCLUDE [Region]: SUM([Sales])}
